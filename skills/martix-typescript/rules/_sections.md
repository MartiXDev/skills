# Rule section contract

Keep each TypeScript rule focused on one decision area and use this order:

1. `## Purpose`
2. `## Default guidance`
3. `## Decision branches`
4. `## Review checklist`
5. `## Related files`
6. `## Source anchors`

Put normative guidance in rules. Put dated facts, compatibility status, and
navigation in references so each meaning has one source of truth.
