# Rule section contract

Keep each Fluent UI rule focused on one decision area and use this order:

1. `## Purpose`
2. `## Default guidance`
3. `## Decision branches`
4. `## Review checklist`
5. `## Related files`
6. `## Source anchors`

Put normative guidance in rules. Put dated package versions, release channels,
peer ranges, and framework compatibility notes in references.
