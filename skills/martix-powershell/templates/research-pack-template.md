# Research pack template

## Purpose

Use this template when preparing a research memo that feeds future
PowerShell cmdlet rules, reference maps, or a later expansion pass.

## Recommended outline

1. Scope, non-goals, and source priority
1. Approved workstream alignment
1. Candidate file inventory
1. Top-level quick-reference candidates
1. Deep-rule candidates by planned prefix
1. Official source list with descriptive links
1. Gaps, open questions, and later follow-up work

## Review prompts

- Which Microsoft PowerShell-Docs pages are authoritative for the topic?
- Which guidance belongs in rules versus future reference maps?
- Which topics are ready for first-pass rules and which stay backlog material?
- Which cross-cutting concerns need explicit review: parameter validation,
  pipeline processing, error handling, confirmation safety, or module
  packaging?
