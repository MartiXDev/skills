## Rule authoring template

### Purpose

Use this template when adding a new rule file under `rules/` so the library
keeps a stable section order and review shape.

### Front section

- Start with an H2 title that matches the rule topic.
- Keep the introductory sentence narrow and decision-oriented.

### Body outline

1. `### Purpose`
1. `### Default guidance`
1. `### Avoid`
1. `### Review checklist`
1. `### Related files`
1. `### Source anchors`

### Drafting prompts

- What should an agent do first when this rule is relevant?
- Which anti-patterns are common enough to warn about immediately?
- Which sibling rules or maps should be linked for deeper context?
- Which official docs prove the recommendation?
